# Bitácora de Desarrollo del Proyecto Xlerion Ultimate

## Resumen General de Avances

Este documento detalla las acciones y modificaciones realizadas en el proyecto Xlerion Ultimate hasta la fecha.

### 1. Configuración Inicial y Estructura del Proyecto
- **Identificación de Tecnologías:** Next.js (App Router), TypeScript, React, Supabase, Trigger.dev.
- **Evaluación de Estado:** Análisis inicial del proyecto basado en `arch.txt` y `package.json`.
- **Creación de Estructura Base:**
    - `src/app/layout.tsx`: Layout raíz de la aplicación.
    - `src/app/page.tsx`: Página de inicio.
    - `src/app/globals.css`: Estilos globales.
    - `src/components/`: Directorio para componentes reutilizables.
    - `src/components/MenuBar.tsx`: Componente de la barra de navegación.
    - `src/components/Footer.tsx`: Componente del pie de página.
    - Páginas de marcador de posición para todas las secciones principales (`sobre-nosotros`, `servicios-y-productos`, etc.) basadas en `documentoEmpresa.txt`.
    - Contenido inicial de todas las páginas poblado con información de `documentoEmpresa.txt`.

### 2. Transición de Estilos: Tailwind CSS a Bootstrap
- **Intento Fallido con Tailwind CSS:** Se encontraron problemas con la instalación y configuración de Tailwind CSS (`ETARGET` error, `npx` ejecutable no encontrado).
- **Migración a Bootstrap:**
    - Desinstalación de `tailwindcss`, `postcss` y `autoprefixer`.
    - Eliminación de `tailwind.config.js` y `postcss.config.js`.
    - Eliminación de directivas de Tailwind de `src/app/globals.css`.
    - Instalación de `bootstrap` y `react-bootstrap`.
    - Importación de `bootstrap/dist/css/bootstrap.min.css` en `src/app/layout.tsx`.
    - Reescritura de `src/components/MenuBar.tsx` para usar componentes `Navbar` de React-Bootstrap.
    - Eliminación de clases de Tailwind de `src/app/page.tsx`.

### 3. Mejoras de Interfaz y Experiencia de Usuario
- **Manejo de Componentes Cliente/Servidor:**
    - Adición de `"use client";` a `src/components/MenuBar.tsx` y `src/app/page.tsx`.
    - Creación de `src/components/ClientNavbar.tsx` para encapsular componentes de React-Bootstrap.
    - Importación dinámica (`dynamic` con `ssr: false`) de `MenuBar`, `Footer`, `FontSizeAdjuster` y `ParallaxVideoScroll` en `src/app/layout.tsx` para asegurar el renderizado del lado del cliente.
- **Replicación de Diseño (basado en `colombiatechfest.ai-hackathon.co`):**
    - **Estilos Globales:** Fondo negro muy oscuro y color primario azul brillante (`#00FFFF`) definidos en `src/app/globals.css`.
    - **Tipografía:** Implementación de Google Fonts (Inter para cuerpo, Montserrat para encabezados) y ajuste de tamaños/pesos de fuente en `src/app/page.tsx` y `globals.css`.
    - **Sección Parallax:**
        - Implementación de un efecto parallax de video de fondo (`ParallaxVideoScroll.tsx`).
        - Adición de botones de Llamada a la Acción (CTA) en la sección parallax de la página de inicio.
    - **Tarjetas (Misión, Visión, Valores):**
        - Adición de imágenes de marcador de posición.
        - Implementación de efectos hover (`card-hover-effect`).
        - Redondeado de una esquina de las tarjetas (`card-rounded-corner`).
        - Truncamiento de texto en las tarjetas para una longitud consistente.
        - Implementación de un modal para mostrar el contenido completo al hacer clic en "Ver más".
        - Estilización de la imagen dentro del modal para que sea más grande y consistente.
    - **Barra de Navegación:**
        - Propiedad `sticky-top` para que el menú se fije al hacer scroll.
        - Línea animada "láser" debajo del menú.
    - **Modal:**
        - Efecto de borde brillante animado alrededor de todo el modal (`modal-glowing-border`).
- **Accesibilidad/Usabilidad:**
    - Implementación de un botón flotante para ajustar el tamaño de la fuente (`FontSizeAdjuster.tsx`).
    - Establecimiento del tamaño de fuente predeterminado a 12px.

### 4. Problemas Pendientes y Recordatorios
- **Marcador de Posición `xlerionml_service`:** Se ha identificado que el comando real para iniciar el servicio ML de Python es `cd ../xlerion-ml && uvicorn main:app --reload`. Por lo tanto, el comando para iniciar todos los servidores (Next.js y el servicio ML) concurrentemente es `npm run dev:all`. Esto resuelve el bloqueo del script `dev:all`.
- **Carga de Imágenes/Video:** Los errores `net::ERR_NAME_NOT_RESOLVED` para las imágenes y el video de marcador de posición son problemas relacionados con la red del usuario. Es crucial verificar la conexión a internet, la configuración de DNS o posibles bloqueos de firewall/proxy. Las URLs de marcador de posición deberán ser reemplazadas por activos reales alojados de forma fiable.

**Comandos para Iniciar Servidores:**
- **Frontend (Next.js):**
    - Navegar al directorio `xlerion-ultimate`: `cd xlerion-ultimate`
    - Iniciar servidor de desarrollo: `npm run dev` o `yarn dev`
- **Backend (FastAPI - asumiendo `xlerion-ml` es el backend):**
    - Navegar al directorio `xlerion-ml`: `cd xlerion-ml`
    - Iniciar servidor FastAPI: `uvicorn main:app --reload` (asumiendo `main.py` es el punto de entrada y `app` es la instancia de FastAPI)

## Comandos de Desarrollo

Aquí tienes un resumen de los comandos para ejecutar el proyecto:

### Iniciar solo el frontend (Next.js)
Este comando inicia la página web.
```
npm run dev
```

### Iniciar solo el backend (Python)
Este comando inicia el servicio de Python que proporciona los datos del menú.
```
npm run xlerionml_service
```

### Iniciar ambos (frontend y backend) al mismo tiempo
Este es el comando recomendado para el desarrollo, ya que inicia todo lo que necesitas con un solo comando.
```
npm run dev:all
```
